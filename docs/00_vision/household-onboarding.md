# Household Onboarding

This document defines the first-run experience of DomusMind.

The onboarding flow must create a **functional household system in minutes**.

The goal is immediate value.

---

# Onboarding Principles

The first-run experience must:

- minimize user input
- avoid technical concepts
- create immediate usefulness
- establish the household model

The system should feel ready within a few steps.

---

# Step 1 — Start Household

The first screen contains one action.

```

Welcome to DomusMind

Your household coordination system.

[ Start your household ]

```

No options.

No configuration.

---

# Step 2 — Name the Household

The system asks:

```

What should we call your household?

[ Carmona Family ]

```

Press **Create Household**.

Behind the scenes:

```

Create household
Create first user
Assign ownership
Initialize system

```

Users never see these operations.

---

# Step 3 — Add People

The system asks:

```

Who lives here?

```

Example:

```

Juan
Gema
Juanito
Julia
Manila

```

Minimal information required.

Optional classification:

```

Adult
Child
Pet

```

This creates the set of people the household coordinates.

---

# Step 4 — Connect Devices

The system asks:

```

Does this person use DomusMind on their own device?

```

Possible actions:

```

This computer
Send invite link
No device

```

Invites may use:

- QR code
- local network link
- manual sharing

The goal is to connect people to the system without friction.

---

# Step 5 — Choose First Area

The system asks what the household wants help with first.

```

What would you like DomusMind to help with?

[ Chores ]
[ Calendar ]
[ Groceries ]
[ Reminders ]

```

Selecting one activates the corresponding household feature.

---

# Step 6 — Immediate Automation

Example: Chores.

DomusMind proposes typical household responsibilities.

```

Trash
Laundry
Dishwasher
Vacuum
Bathroom

```

The household selects which apply.

Then the system asks:

```

Rotate chores automatically?

```

Once confirmed, DomusMind begins assigning responsibilities.

The household immediately sees results in the **Household Timeline**.

---

# Onboarding Outcome

At the end of onboarding the household should have:

- people registered
- first responsibilities configured
- timeline populated
- coordination working

The system should already feel useful.

Users should feel that the household has gained **structure and clarity**.
