# Chore Assignment

DomusMind removes the negotiation around household chores.

Instead of families constantly asking:

> Who will do this?

DomusMind automatically assigns work so the household runs smoothly.

The goal is not task tracking.

The goal is **predictable household operation**.

---

# Concept

Households contain recurring responsibilities:

- trash
- dishwasher
- laundry
- cleaning
- small maintenance

These responsibilities must happen regularly, but coordinating them manually creates friction.

DomusMind manages these responsibilities automatically.

The system determines **who is responsible today**.

---

# Household Chores

A chore represents a recurring household responsibility.

Examples:

```

Trash
Dishwasher
Laundry
Vacuum
Bathroom cleaning

```

Chores exist to ensure the household keeps running.

They are not treated as a productivity system.

---

# Assignment Philosophy

The household should not spend time negotiating chores.

DomusMind automatically distributes responsibilities using simple rules.

Typical patterns include:

### Rotation

Responsibilities rotate between people.

Example:

```

Mon → Juan
Tue → Lucía
Wed → Juan
Thu → Lucía

```

This distributes recurring work fairly.

---

### Default Owner

Some chores naturally belong to one person.

Example:

```

Laundry → Lucía

```

The responsibility is still visible to the household.

---

### Shared Responsibility

Some tasks can be completed by anyone.

Example:

```

Take recycling downstairs

```

These appear as open household tasks.

---

# Daily Experience

Users do not open a “chore manager”.

Instead, chores appear naturally in the **Household Timeline**.

Example:

```

Today

Trash → Juan
Dishwasher → Lucía

```

Completion requires a single action.

```

✓ Done

```

The goal is minimal friction.

---

# Missed Chores

When a chore is missed, the system quietly surfaces the situation.

Example:

```

Trash was not completed yesterday

```

DomusMind may suggest a correction:

```

Reassign to Lucía?

```

This prevents frustration and blame between household members.

---

# Household Visibility

Everyone can see the current state of responsibilities.

This improves:

- fairness
- transparency
- coordination

No one needs to ask:

> Who was supposed to do this?

---

# Relationship to the Timeline

Chores are not a separate system.

They simply appear in the **Household Timeline** alongside:

- plans
- tasks
- reminders

This ensures the household sees everything in one place.

---

# Design Goal

Chores should feel like **background infrastructure**.

The system quietly ensures responsibilities are distributed and visible.

Household members should not think:

> I need to manage chores.

They should simply know:

```

What is my responsibility today?

```
