# DomusMind — Public Marketing Foundation

## Purpose

This document defines the public-facing messaging foundation for DomusMind.

It exists to support:

- homepage copy
- landing pages
- SEO pages
- onboarding copy
- product descriptions
- social posts
- app store text
- supporter and tester recruitment

This is not a technical document.

This is the messaging source of truth for public communication.

---

## Language Rule

Public language must be close to the household reality of the user.

Use:

- household
- people
- plans
- chores
- tasks
- today
- who is responsible
- what matters
- what needs attention

Avoid:

- operational state
- bounded context
- domain model
- coordination infrastructure
- execution layer
- accountability model
- aggregate
- projection

Public messaging must sound like lived life, not software architecture. This aligns with the product’s own UX rule that household language must remain visible while internal model language stays hidden. fileciteturn1file2

## Mission

DomusMind exists to remove the invisible mental load of running a household.

It turns scattered plans, chores, and responsibilities into one shared system so the home no longer depends on one person remembering everything.

This directly reflects the product mission to eliminate invisible cognitive load and transform it into a shared operational system. fileciteturn1file5

DomusMind helps households stop depending on one person’s memory.

## Positioning Statement

For couples and families who are tired of carrying household coordination in their heads, DomusMind is a household coordination system that makes plans, chores, and responsibilities visible in one shared place.

Unlike calendars, to-do apps, or chat threads, DomusMind is built for the reality of home life: shared responsibility, recurring chores, family schedules, and the daily question of what matters today.

## Core Problem

Most households do not run on a shared system.

They run on a fragile mix of:

- memory
- chats
- notes
- separate calendars
- mental lists
- repeated reminders

This creates a hidden imbalance:

- one person remembers everything
- others wait to be told
- responsibilities stay implicit
- plans and chores remain fragmented
- things are forgotten until they become conflict

The real problem is not lack of effort.

The real problem is lack of shared visibility.

## Copy Rules

### Must Do

- speak in household language
- lead with pain and outcome
- make the reader feel seen fast
- use concrete daily examples
- keep the whole message understandable in five seconds
- show difference through real-life situations

### Must Not Do

- explain the architecture
- use software jargon
- sound like a B2B SaaS tool
- promise AI magic you cannot deliver
- say “all-in-one” without substance
- say “organize your life”
- speak too broadly

## Final Messaging Test

Every public sentence should pass these checks:

1. Would a tired parent or overloaded partner understand it instantly?
2. Does it sound like home life, not software design?
3. Does it make the pain visible?
4. Does it make the outcome desirable?
5. Can the product actually deliver it?

If not, rewrite it.
