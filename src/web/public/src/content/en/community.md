DomusMind is built in the open because household coordination should be transparent, inspectable, and owned by the people it serves.

## For families

You do not need to write code to help.

If you recognize the problem — one person keeping everything in their head — your experience matters.

- Share what coordination in your house actually looks like
- Tell us where your current tools fall short
- Follow the project and become an early adopter when V1 is ready

[Follow on GitHub](https://github.com/juangcarmona/domusmind)

## For developers

DomusMind has a real domain and a real architecture.

If you want to contribute to a problem that is structural, not cosmetic, there is meaningful work here:

- Implement V1 capabilities: calendar, tasks, routines
- Improve the domain model and ubiquitous language
- Extend the API or build integrations
- Improve documentation, translations, or accessibility

[Explore the architecture](/architecture) · [View specs on GitHub](https://github.com/juangcarmona/domusmind/tree/main/specs)

## Contribution paths

- **GitHub issues** — bug reports, questions, proposals
- **GitHub discussions** — architecture ideas, domain feedback, roadmap input
- **Pull requests** — code, docs, translations

All contribution happens at [github.com/juangcarmona/domusmind](https://github.com/juangcarmona/domusmind).

## Roadmap

**V1 — Core system** _(in progress)_
Family structure, responsibilities, events, tasks, routines, timeline.

**V1.1 — Shared lists**
Persistent shared checklists for household coordination.

**Extended model** _(future)_
Properties, food, pets, administration, and intelligent assistance.
