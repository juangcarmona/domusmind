## Why this cannot be a closed tool

The house should not depend on a company's servers.

A household coordination system holds real family data.
Who is responsible for what.
Where the children need to be.
What is happening at home.

That data belongs to the family.
Not to a SaaS company.
Not to an app that can disappear, change its pricing, or sell the data.

## Why families need control

When one person already carries the house in their head, the last thing a family needs is to also depend on an opaque external system.

A closed tool creates a new single point of failure.

If the service goes down - coordination breaks.
If the app changes - the family has to adapt.
If the company disappears - the data is gone.

Families deserve a system they can own, inspect, and keep.

## Why open source matters here

Open source is not a philosophical position.

It is a practical requirement for a system families can trust.

- The code is readable. You can verify what it does with your data.
- The system is self-hostable. You can run it on your own infrastructure.
- The model is extensible. You can adapt it to how your household works.
- The community can improve it. Bugs get fixed. Features get added.

This is not about ideology. It is about reliability and trust.

## How developers can help

DomusMind is a real product with a real domain.

There is meaningful work here:

- Implementing new V1 capabilities (calendar, tasks, routines)
- Improving the domain model
- Extending the API
- Building integrations (notifications, mobile clients, voice)
- Improving the UI and accessibility
- Translating and localizing

If you believe that household coordination should be fair, shared, and transparent - you can help build the system that makes it possible.

## Connection to fairness

Responsibility can only be shared when it is visible.

DomusMind makes household state visible.
Open source makes the system itself visible and trustworthy.

Both matter.
